使用hexo搭建GitHub博客

作为一个新人深深的感觉到自己的技术有多浅薄 接触的越深越能感觉到学海无涯 所以就决定重新开始写博客
之前也在其他的网站上写过一点东西 那时候给我的感觉就为了写东西而写 从各位前辈那里搬几块砖 实现了功能就觉得不错了 其实并没什么卵用 现在希望扎扎实实的学习

第一篇博客就记录一下这个博客的搭建过程 和具体一些使用方法

搭建步骤

  1. 安装GIt
  2. 安装Node.js
  3. 安装Hexo
  4. Hexo本地网站与Github绑定
  5. Hexo设置与Next样式的使用
  6. 写写写文章 以及常用命令

安装Git

Windows下载地址：https://git-scm.com/download/win
官网的git教程：https://git-scm.com/book/zh/v2
安装git不需要什么注意的默认就好 安装之后需要配置用户信息 官方的教程 还有网上的N多教程都有
我对git的使用仅限于几个常用的命令 就不过多的阐述了

安装Node.js

Windows下载地址 ：https://nodejs.org/en/
默认安装即可
安装Node.js会包含环境变量及npm的安装 安装完成之后 打开命令行 （使用git bash也是可以的）
输入 node-v 检测Node.js 输入npm-v 检测npm 都会输出版本号
安装完Git和Node.js之后基础的环境就搭建完成了

安装Hexo

打开命令行 使用 npm 安装 Hexo 执行完命令之后需要做的就是慢慢等待

安装完成之后 新建一个文件夹 这里就叫blog 进行建站 这个过程也非常简单

等待这个过程结束之后 建站也就完毕 在blog文件中会自动创建一个 folder 你输入的名字 一切有关这个网站的操作都在 此folder文件中打开命令行进行操作

现在呢 建站已经完毕了 看看效果 开打命令行 输入 hexo s 然后会提示你
在浏览器中输入 http://localhost:4000/ 就可以看到网站 这里需要注意的是 后面一句话 Press Ctrl+C to stop 按下Ctrl+c会停止服务 不得不说这个设定很是可以 我最开始的时候就习惯的去copy结果发现网站打不开 懵逼了好一会儿
如果忘记Ctrl+c去停止服务也没关系 命令行关闭 服务自动停止

Hexo本地网站与Github绑定

首先呢需要在github上new repository 新建这个仓库需要注意的是
仓库名的格式必须是 【你的用户名.github.io】 错一个字都不行
选择新创建的仓库 选择setting 在设置页面找到GitHub Pages模块
这样我们github端的操作就完成了

安装 hexo-deployer-git

打开命令行 安装hexo部署到git的插件

在本地文件夹中找到 _config.yml 这个文件是配置文件
打开文件夹 在最下面找到 deploy 进行配置 配置的时候需要注意： 后面要加一个空格

保存文件 打开命令行 第一次部署到服务端的时候 会验证用户名和密码 成功之后 访问http://xxx.github.io 就可以访问到网页了 不得不说 简单又漂亮 科技改变生活 感谢前辈们 哈哈
如果没有立即生效可以等一小会儿 你会发现你的本地网页 也被自动提交到GitHub上了

主题使用呢推荐使用 next 这个主题已经非常完善了 有官方的文档使用很方便 只需要专心写文章就可以了 很多人都在用

把主题从github上clone到你的 theme文件夹中 然后 在主文件夹的_config.yml 找到 theme 配置一下就可以 记得运行一下命令 就可以查看效果了

写文章

写文章呢 语法格式用markdown 我们用命令就可以创建一篇文章 md文件在source下的_posts文件夹下

打开生成的md文件 会发现默认生成了一些对于文章的配置 在这个配置模块就可以对文章进行分类 而且这个分类也不需要预先设置好 只要在文章中设置好 hexo就会给你自动生成一切有关的东西

layout 布局
title 标题
date 建立日期 文件建立日期
updated 更新日期 文件更新日期
comments 开启文章的评论功能 true
tags 标签（不适用于分页）
categories 分类（不适用于分页）
permalink 覆盖文章网址
